# Phase 2 Plan: Landing Content

> "Was wir bieten" + "Für wen" Sections

## Goal

Statische Content-Sections die erklären was Generation AI bietet und für wen es ist.

## Tasks

### Task 1: Features Section Component
**File:** `components/sections/features.tsx`

- 4 Feature Cards im Grid:
  1. **KI-Tool-Bibliothek** — Kuratierte Tools für Studierende
  2. **Community** — Austausch mit anderen KI-Interessierten  
  3. **Kurse & Guides** — Lerne KI praktisch anzuwenden (Coming Soon)
  4. **Events** — Webinare, Workshops, Guest Speaker (Coming Soon)
- Icons für jede Card (Lucide Icons)
- Responsive: 1 col mobile, 2 col tablet, 4 col desktop
- Section ID: `#features` (für "Mehr erfahren" Button)

### Task 2: "Für wen" Section Component
**File:** `components/sections/target-audience.tsx`

- Headline: "Für Studierende im DACH-Raum"
- Subline: Value Proposition (kostenlos, Community, praxisnah)
- Optional: 2-3 kurze Use Cases oder Personas
- Visuell ansprechend, passend zum Design System

### Task 3: Integration in Homepage
**File:** `components/home-client.tsx`

- Features Section nach Hero einfügen
- "Für wen" Section nach Features
- Scroll-Anchor `#features` funktioniert
- Spacing zwischen Sections

### Task 4: Mobile Responsive Check
- Alle Sections auf Mobile testen
- Breakpoints: sm (640px), md (768px), lg (1024px)
- Touch-friendly Abstände
